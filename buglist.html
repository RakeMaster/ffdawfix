Список багов, найденных в HTML,JavaScript,CSS в ходе исправлния отображения под Firefox
document.all
Данная функция придумана MS. В стандарте ее не существует.

У себя вставил такую вот затычку:
	if(!document.all) {
		document.all = function(id,index) {
			return document.getElementById(id);
		}
	}
В идеале нужно везде заменить. Работоспособность под IE должна остаться той же
innerText
Данная проперти придумана MS. В стандарте ее не существует.

У себя вставил такую вот затычку:
	HTMLElement.prototype.__defineGetter__("innerText", function () {
	    return this.textContent;
	});
	HTMLElement.prototype.__defineSetter__("innerText", function (val) {
		this.textContent = val;
	});
Если для FF ее воткнуть в начале страници, то работать должно.
В идеале нужно везде заменить на textContent(правда не знаю есть ли в IE)
showModelessDialog
Данная функция придумана MS. В стандарте ее не существует.

У себя вставил такую вот затычку:
if(!window.showModelessDialog)
	window.showModelessDialog = function(url, loc, p){
		p = p.replace(/dialog/g,"");
		p = p.replace(/;/g,",");
		p = p.replace(/:/g,"=");
		p = p.toLowerCase();
		window.open(url,"",p);
	}
По мне лучше переписать на window.open. Но можно и так. Эта функция используется для
открытия окна смайликов
ALT,TITLE
ALT - альтернативный текст, показывающися вместо картинки (когда они отключены или недоступны)
TITLE - текст, всплывающий при наведении на картинку

Разработчики IE решили, что будет здорово иметь TITLE=ALT в случае если ALT есть, а TITLE нет
Я писал скрипт, опбегающий дом дерево и проставлящий TITLE
Но нужно везде добавить недостающиее свойство TITLE
Проблем от этого много (подарки, награды, значки кланов ...)
cursor: hand 
Нету курсора hand. Есть курсор pointer
Нужно в коде и в CSS заменить hand на pointer.
Должно по идее работать везде

Firefox обрезает неизвестное значение hand и вместо лапки мы видим стрелочку
document.frames
	У документа нету frames.
	Я заткнул так
		document.frames = window.frames;
	Соответственно можно кросбраузерно так
		if(!document.frames) document.frames = window.frames;
html background-color
	Из за указания в CSS цвета фона на таг HTML (помимо body)
	на Firefox (и кажется опера) на многих локациях и в списке чата фон выглядит обрезнным
	Исправляется так
		html {
			background-color: transparent;
		}
	Или лучше просто убрать цвет фона с html и оставить только на body
Точки на карте
	На фрейме с картой (окрестности) неправильно задан список аттрибутов в style
	Пример
		<img src="/vr/maps/self.gif" style="WIDTH: 8px; POSITION: relative; HEIGHT: 8px; ZORDER: 102; TOP=210; LEFT=240; ">
		<img src="/vr/maps/self-end.gif" style="WIDTH: 8px; POSITION: relative; HEIGHT: 8px; ZORDER: 101; TOP=0; LEFT=0; DISPLAY: none; ">
	Внимание нужно обратить на TOP и LEFT. Они написаны через "=", хотя должно быть ":"
	Незнаю почему IE это ест, но Firefox не хочет (а он прав!)
	Замена быстрая и работоспособность дожна остаться везде
	Кстати неплохо бы указать размерности (px)
dialogArguments
	На фрейме со смайлами нету dialogArguments
	Можно закрыть так
	if(!window.dialogArguments)	window.dialogArguments = opener.top;
chouter
	На Layout неправильно описан фрейм chouter
		<frame id="chouter" src="Cht/ChOuter3.htm?7" marginwidth="0" scrolling="no">
	Должно быть не id а name="chouter"
	Из за этого Firefox не может найти фрейма при обращении через frames
	Если исправить должно работать везде
